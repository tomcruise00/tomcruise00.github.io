Ubuntu下安装Nextcloud

文件服务器，是一个公司最常用的服务应用，每个公司企业基本都有自己的文件服务器实，现储存分享，上传下载文件文档等功能，常见的文件服务器就是ftp服务器，但是ftp服务器的功能实在有限，且对于普通用户使用入手难度较大，又缺乏界面，对于公司的普通用户来说，确实不是一个好的文件服务器。一般ftp也只是用户服务器，网站应用等方面。

在目前的公司企业环境中，企业网盘则是一个更好的文件服务器替代方案。百度云盘，相信大家基本都有用过吧，友好的界面交互，网页端，客户端都有，不需要任何命令，实现上传下载，分享等诸多功能。但是这种云盘毕竟不是自己的，很多公司对于将机密文件放在上面心存疑惑，且时不时的网盘关闭热潮也让人担心。既然如此，为何不搭建自己的网盘呢，于是就有了nextcloud。

对于私人网盘，市面上已经有很多的产品，很多开源半开源的云盘系统。其中最出名的就是 seafile和owncloud/nextcloud。seafile是国人开发的，分块处理，断点上传，速度比后者要快。有社区版和企业版。社区版免费，但是功能有限，企业版要收费，功能更强大。但是社区版的功能，老实说只能满足个人使用，无法满足企业使用。而企业版要收费，费用根据公司人数不同，还需要发邮件询问。

像我这种人数不多，又不想花钱的，那么nextcloud就是最好的选择，nextcloud是owncloud的一个分支，由原创始人团队维护，是在owncloud被别的公司收购后，由创始人团队创立的新分支。就像 mysql和mariadb。nextcloud完全开源，功能强大，是外国人开发维护的。具体的与seafile等的对比，这里不详细说明了，有兴趣的百度就好。下面记录一下在ubuntu服务器上搭建nextcloud的具体过程。

自用网盘，没有域名IP访问，修改了默认端口，开启https，实验很多次，下面是过程，基本每一步都有了

首先先

sudo apt-get update

基础环境安装（LAMP）

nextcloud的安装基础环境有两种，一种LAMP，一种LNMP，我选择了LAMP，区别就是一个用的Apache一个用的Nginx.

安装 Apache2

sudo apt install -y apache2

在浏览器输入你的IP，能看到apache2的网页代表安装成功
安装完成后启用下面这些模块，为了防止后面出问题，这边先启用

sudo a2enmod rewrite dir mime env headers
service apache2 restart

以下是一些Apache2的说明
1.修改状态：service apache2 status|start|stop|restart
2.站点目录：/var/www/
3.修改权限：sudo chmod 777 /var/www -Rf #这个比较重要，后面会讲到
4.安装目录：/etc/apache2/
5.全局配置：/etc/apache2/apache2.conf
6.监听端口：/etc/apache2/ports.conf
7.虚拟主机：/etc/apache2/sites-enabled/000-default.conf

运行命令

sudo vim /etc/apache2/apache2.conf

在该文件的最后面添加

AddType application/x-httpd-php .php .htm .html
AddDefaultCharset UTF-8

然后重启Apache服务。
到此为止，web站点其实已经搭建起来了，只需在/var/www/html目录下存放网页文件，电脑主机就可以访问服务器了，http://ip/xxx.html
为nextcloud配置端口等信息

为nextcloud创建Apache配置文件

sudo vim /etc/apache2/sites-available/nextcloud.conf

新建文件，在里面添加以下内容：

<VirtualHost *:80>
ServerAdmin admin@example.com
DocumentRoot /var/www/html/nextcloud
ServerName example.com
ServerAlias www.example.com
ErrorLog /var/log/apache2/nextcloud-error.log
CustomLog /var/log/apache2/nextcloud-access.log combined
<Directory /var/www/html/nextcloud/>
Options +FollowSymlinks
AllowOverride All
Require all granted
SetEnv HOME /var/www/html/nextcloud
SetEnv HTTP_HOME /var/www/html/nextcloud
<IfModule mod_dav.c>
Dav off
</IfModule>
</Directory>
</VirtualHost>

使用以下命令创建apache2必要的符号链接:

sudo ln -s /etc/apache2/sites-available/nextcloud.conf /etc/apache2/sites-enabled/nextcloud.conf

安装 MariaDB数据库

sudo apt install -y mariadb-server mariadb-client

启动MariaDB服务

sudo systemctl start mysql

查看状态

sudo systemctl status mysql

重启服务

service mysql restart

执行初始化安全脚本，默认root密码为空，设置root密码和其他选项

sudo mysql_secure_installation

为NextCloud创建数据库和用户
创建数据库nextcloud，用户名nextcloud，密码XXXXXXXX

sudo mysql -u root -p

进入MariaDB后，执行以下命令，分号必须带上
创建数据库，名为：nextcloud

CREATE DATABASE nextcloud;

创建用户nextcloud，password可自定义,下面的命令自己改

CREATE USER nextcloud@localhost identified by 'XXXXXXXX';

用以下命令授予必要的权限为数据库和用户授权

GRANT ALL PRIVILEGES ON nextcloud.* TO nextcloud@localhost identified BY 'XXXXXXXX';

用命令更新数据库配置:

flush privileges;
exit;

安装 PHP 新的版本需要安装更高PHP版本https://ywnz.com/linuxjc/3782.html

sudo apt install php

查看PHP版本

php -v

安装 PHP 扩展

sudo apt install -y php7.0-gd php7.0-json php7.0-mysql php7.0-curl php7.0-mbstring php7.0-intl php-imagick php7.0-xml php7.0-zip

可以将7.0改成你对应的PHP版本

service apache2 restart

重启apache2服务
创建一个PHP文件看是否可以正常访问

sudo vim /var/www/html/phpinfo.php

打开文件后在里面加入一下内容

<?php echo phpinfo();?>

保存，然后在浏览器输入以下内容
http://你的ip/phpinfo.php 能看到php网页代表成功

安装phpmyadmin

这个主要是用来用网页管理数据库的

sudo apt-get install phpmyadmin

安装过程选择apache2，系统还会提示您安装数据库。请确保为该选项选择Yes。接下来，将要求您为phpmyadmin设置MySQL应用程序密码（因此它可以向数据库服务器注册）。输入并验证高强度的密码。最后，您必须输入数据库管理员用户密码。
创建phpMyAdmin快捷方式

sudo ln -s /usr/share/phpmyadmin /var/www/html

确保apache2的rewrite功能开启，前面有
最后输入http://你的IP/phpmyadmin看是否有页面
这里可以用上面在MariaDB里面设置的账号密码登录

安装nextcloud

安装（选择稳定版）

cd /var/www/html
sudo wget https://download.nextcloud.com/server/releases/nextcloud-16.0.3.zip
unzip nextcloud-16.0.3.zip
rm nextcloud-16.0.3.zip

具体下载版本可以到这里面找
https://download.nextcloud.com/server/releases/

给文件相应的权限

cd /var/www
sudo chown www-data:www-data html -Rf
sudo chmod 750 html -Rf
cd /var/www/html
sudo chown www-data:www-data nextcloud -Rf
chmod 770 nextcloud -Rf

这里的设置是我找了资料自己理解后设置的，感觉这样设置比较安全，这里www-data是WEB服务的默认用户和用户组，没有直接用root，应为用root的话要给777权限，这样不安全，这样只针对www-data有权限，其他用户没有权限修改网站的根目录

这里就可以登录了，登录第一次会让你设置数据目录和数据库，我的数据目录没有用默认的，是自己新建的文件夹，所以也要设置这个文件夹的用户和权限为www-data和770

生成自验证ssl证书、开启https

参考 https://blog.csdn.net/Sky_qing/article/details/8940420
最新参考 https://blog.csdn.net/Sky_qing/article/details/44303221
https://www.jianshu.com/p/a2a021b5f3db
https://www.songma.com/news/txtlist_i25881v.html

为证书新建个目录

mkdir /etc/apache2/cert

使用openssl生成自签证书（有效期10年），若无openssl请先安装，按提示操作即可

openssl req -new -x509 -days 3650 -nodes -out /etc/apache2/cert/cert.crt  -keyout /etc/apache2/cert/cert.key

修改证书和文件夹权限

sudo chown -R 600 /etc/apache2/cert/*
sudo chown -R 700 /etc/apache2/cert

修改apache配置启用证书和https

sudo vim /etc/apache2/sites-available/default-ssl.conf  //设置文件修改部分，如下面
<VirtualHost _default_:443>
SSLEngine on
SSLCertificateFile  /etc/apache2/cert/cert.crt
SSLCertificateKeyFile /etc/apache2/cert/cert.key

下面启用ssl

sudo a2enmod ssl
sudo a2ensite default-ssl

添加可信域

vim /var/www/html/nextcloud/config/config.php  // 修改其中的，如下
'trusted_domains'=>array(0=>'localhost',1=>'IP:5005',2=>'IP',),

重启apache服务，每次改完网站配置都要重启下

/etc/init.d/apache2 restart

最后登录 https://IP:5005/nextcloud, 忽略安全提示，使用360极速一类浏览器（不要IE8），登录去吧。如有补充，后期完善
